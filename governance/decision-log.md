# Decision history

This is the stable entry point for Whitefoot's historical design and
implementation decisions. The record is split by the kernel-specification era
that was active when the work happened. Cross-version repository decisions are
filed under the active era so the chronology remains intact.

The history explains why earlier work existed. It does not authorize current
work or override `docs/roadmap.md`, the active numbered specification,
`docs/constitution.md`, or `governance/directives.md`.

## Recording rule

Add a new entry to the file for the specification version it concerns. If an
entry is repository-wide, put it under the version active when the decision was
made. Start a new dated volume only when an era file becomes difficult to
review; update this index in the same change. Do not rewrite old entries to
match current terminology.
